# دليل تكامل مطوري الذكاء الاصطناعي والمساعدين الرقميين لإدارة وتصحيح البناء التلقائي 🤖🛠️

يرحب بكم نظام المراقبة الذكية لـ **Marium MEXC**! تم إعداد مستودعكم بالكامل لتطبيق دورة حياة برمجية ذاتية الإصلاح (Self-Healing DevOps Loop). سيوضح لكم هذا الدليل كيفية ربط المساعدين البرمجيين وأدوات الذكاء الاصطناعي المجانية لمراقبة عمليات التجميع (Builds)، وتصحيح الأخطاء تلقائياً فور حدوثها دون تدخل بشري.

---

## 🧭 آلية عمل نظام الأتمتة والتشخيص الذاتي المدمج

لقد قمنا بإنشاء وتفعيل ملف تدفق العمل (GitHub Actions Workflow) في `.github/workflows/build.yml` والذي ينقسم آلياً إلى المراحل التالية:
1. **التجميع التلقائي (Automatic Build):** يقوم ببناء ملف الـ APK بشكل آلي ومستمر عند كل دفع للكود (`push`) أو طلب سحب (`pull request`).
2. **الاستخلاص والتحليل الفوري (On-Failure Diagnostics):** في حال حدوث أي خطأ برمي (Compiler Error)، يقوم النظام باستخلاص الأسطر المعطوبة مباشرة من سجل التجميع وكتابة تقرير تشخيصي باللغة العربية داخل ملخص الخطوة في GitHub (Job Summary).
3. **توليد التقارير الذكية (Automated Issue Creator):** يقوم النظام بفتح تذكرة خطأ (GitHub Issue) جديدة مصممة كـ "موجه ذكاء اصطناعي جاهز" (Prompt) يتضمن الأسطر المعطوبة بالضبط وتعليمات واضحة للمساعدين الرقميين لكيفية حلها.
4. **تنبيه المساعدين الخارجيين (Webhook Dispatching):** إرسال تنبيه فوري يحتوي على بيانات الخطأ إلى أدوات الأتمتة الخارجية في حال ربطها.

---

## 🔗 1. ربط المساعدين الخارجيين عبر الـ Webhooks (تنبيه فوري للأخطاء)

يمكنكم استقبال تنبيهات التجميع وتفاصيل الأخطاء فوراً على منصات التواصل أو أدوات الأتمتة (مثل Discord، Slack، أو منصات n8n و Make.com) باتباع الخطوات التالية:

1. قم بإنشاء رابط ويب هوك (Webhook URL) من القناة المفضلة لديك (مثلاً في Discord: الإعدادات -> التكامل -> Webhooks).
2. اذهب إلى مستودعك على GitHub.
3. توجه إلى **Settings** (الإعدادات) -> **Secrets and variables** -> **Actions**.
4. اضغط على **New repository secret** (سر مستودع جديد).
5. قم بتسميته بالضبط: `AI_ASSISTANT_WEBHOOK` وضَع الرابط المستنسخ في خانة القيمة.
6. ستقوم الأتمتة الآن بإرسال تقرير مفصل عند كل نجاح أو فشل لعملية البناء.

---

## 🤖 2. ربط مساعدين رقميين مجانيين للإصلاح التلقائي (Auto-Fixing loop)

للحصول على بيئة تطوير ذاتية الإصلاح بالكامل (أي يكتشف الذكاء الاصطناعي الخطأ، يصححه، ثم يدفع الكود من جديد!)، نوصي بالطرق المجانية والذكية التالية:

### الخيار أ: استخدام أدوات الأتمتة المجانية (n8n.io / Make.com) مع Gemini API المجاني
يمكنكم بناء تدفق أتمتة بسيط ومجاني تماماً يقوم بالتالي:
1. **المحفز (Trigger):** مراقبة تذاكر الأخطاء الجديدة (Issues) في مستودع GitHub التي تحمل وسم `build-failed`.
2. **التحليل (AI Prompting):** إرسال محتوى التذكرة (الذي صممناه ليكون موجه جاهز وممتاز) مع ملف الكود التالف إلى نموذج **Gemini 2.5 Flash** (المجاني عبر Google AI Studio).
3. **التنفيذ والكتابة (Auto-Commit):** يقوم الذكاء الاصطناعي بتوليد الكود المصحح، ثم تقوم أداة n8n بكتابة التعديل ودفعه مباشرة إلى GitHub عبر الـ API.
4. **إعادة التشغيل:** عند دفع التعديل، سيقوم تدفق العمل بإغلاق التذكرة تلقائياً وإعادة بناء التطبيق لإنتاج الـ APK بنجاح!

### الخيار ب: استخدام المساعدين البرمجيين المدمجين في متجر GitHub (GitHub Marketplace)
هناك العديد من المساعدين المجانيين الجاهزين للتنصيب في مستودعك بضغطة زر واحدة:
- **Sweep AI أو CodiumPR:** مساعدون رقميون يتفاعلون مباشرة مع الـ Issues. عند فتح النظام لتذكرة أخطاء التجميع `build-failed`، سيقوم Sweep آلياً بقراءتها، إنشاء فرع جديد (Branch)، وتطبيق التعديلات البرمجية لحل المشكلة وتقديم طلب سحب (PR) مصحح بالكامل.
- **GitHub Copilot Workspace:** (متاح للمطورين) يتيح تفعيل معالجة الأخطاء بنقرة واحدة بمجرد قراءة تقرير التشخيص التلقائي المكتوب في الـ Issue.

---

## 🚀 3. استدعاء وبناء التطبيق يدوياً ومراقبته (Workflow Dispatch)

لقد قمنا بتوفير ميزتي:
* **Workflow Dispatch:** تتيح لكم أو لأي مطور مساعد تشغيل عملية البناء يدوياً من واجهة GitHub Actions دون الحاجة لدفع كود جديد.
* **Repository Dispatch:** تتيح لأي تطبيق ذكاء اصطناعي خارجي أو أداة أتمتة (مثل n8n) إرسال إشارة برمجية (API Call) لتشغيل التجميع واختبار سلامة الكود.
